# Release Notes - 2026-05-28

These notes describe the 2026-05-28 public release preparation state for
`altium-cruncher` package version `2026.5.28`.

## Release Status

This release is intended to make the command-line workflows available for public
testing and integration. Users should expect that some commands may still have
errors, omissions, undocumented edge cases, or output differences from specific
Altium versions and project styles.

Please report issues with the exact command, version output, expected behavior,
actual behavior, and a public-safe reproduction fixture when possible.

## Highlights

- Cross-platform `altium-cruncher` CLI for Altium document workflows.
- Schematic and PCB SVG export commands.
- PCB layer STEP export for fixture-alignment models.
- BOM, pick-and-place, JLCPCB, design JSON, extraction, split, merge, project
  decomposition, and cleanup commands.
- JSONC configuration support for config-driven commands, including generated
  default configs with option comments.
- Public command design documentation and contract-focused tests for stable
  configs and machine-readable outputs.
- Windows PowerShell installer wrapper for users who are not already using
  Python or `uv` tooling directly.
- GitHub Actions CI for pull requests and `main` branch pushes on Ubuntu and
  Windows.

## Command Overview

- `version`: prints `altium-cruncher` and controlled dependency versions.
- `sch-svg`: generates schematic SVG from SchDoc, PrjPcb, or SchLib inputs.
- `pcb-svg`: generates PCB SVG views from PcbDoc or PrjPcb inputs.
- `pcb-layer-step`: generates a colored STEP model for one PCB layer.
- `svg`: dispatches schematic SVG, PCB SVG, or both based on input.
- `bom`: generates BOM output as CSV, JSON, or XLSX.
- `pnp`: generates pick-and-place output as CSV, JSON, XLSX, or JLC CPL.
- `jlc`: generates JLCPCB BOM and CPL outputs.
- `design`: generates design JSON with nets, components, and SVG IDs.
- `extract`: extracts symbols, footprints, or IntLib sources from Altium
  design documents.
- `split`: splits multi-item SchLib or PcbLib documents.
- `merge`: merges multiple SchLib or PcbLib documents.
- `megamaid`: decomposes a PrjPcb into libraries, BOM, netlist, and embedded
  assets.
- `clean`: normalizes SchDoc, SchLib, or PcbLib assets using JSON/JSONC config.
- `easyeda-import`: experimentally generates Altium library artifacts from
  EasyEDA/LCSC component data.

## Beta And Experimental Areas

`pcb-svg` HLR and pin-oriented views are beta quality. The command is useful for
current fixture and documentation workflows, but hidden-line rendering, embedded
STEP projection, pin visibility, and related view details are expected to
improve. Output may contain errors or omissions.

`easyeda-import` is experimental. It generates SchLib, PcbLib footprint, and
downloaded 3D model assets by default when the optional EasyEDA dependency is
installed. 3D model placement into the generated PcbLib is not implemented.
Command behavior, generated artifacts, config, and dependency details may
change.

## Known Caveats

- Public behavior is still being hardened against a wider range of Altium
  projects.
- Generated SVG, STEP, BOM, PnP, and JSON outputs may have incomplete handling
  for some project constructs.
- Config defaults are intended to be readable and reviewed before use.
- EasyEDA workflows require the optional `easyeda` extra and should not be
  treated as stable automation yet.

## Compatibility Notes

For public behavior changes, contributors should open an issue or discussion
first and agree on command behavior, config shape, expected outputs, design
docs, and contract tests before opening a pull request.
