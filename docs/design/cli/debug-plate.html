CLI command design

debug-plate

Generates fixture mating-board plans and runnable MCO files from a DUT PCB selection.

Purpose

debug-plate is the first Altium Cruncher mating-board workflow. It inspects a device-under-test PCB, writes editable JSONC configuration, resolves known fixture parts, emits Monkey Change Order operations, and can execute those operations to create a new Altium project. The Cricket Node and node-test-array assets are the reference case.

The current command remains Altium-specific. The config shape is being kept portable enough to later map to KiCad and generic PCB Cruncher workflows; the draft portable shape is documented in debug-plate-mate-config.html.

Usage

altium-cruncher debug-plate init debug-plate.jsonc --force
altium-cruncher debug-plate inspect DUT.PrjPcb --doc board.PcbDoc
altium-cruncher debug-plate seed DUT.PrjPcb --config debug-plate.jsonc
altium-cruncher debug-plate parts-cache build node-test-array.PrjPcb
altium-cruncher debug-plate plan debug-plate.jsonc --output-mco debug-plate.mco.jsonc --force
altium-cruncher debug-plate run debug-plate.jsonc --emit-mco debug-plate.mco.jsonc --force

Arguments

AreaContract
sourceIdentifies the DUT project or PCB document. The command extracts positions, source layers, nets, component footprints, free pads, and board outline information from source files.
outputControls generated project path, filenames, overwrite policy, board outline, and layer-stack template.
known_partsPoints to a known-parts manifest and cache root. The Cricket seed cache maps test points, mounting holes, and free NPTH alignment pads to fixture symbols and footprints.
placementDefines source mount side, mirroring, mirror origin, and output offset in mils. Source top/bottom mixing is rejected for normal component projections.
selectionUses reviewed source objects and projection selectors. Compact designator selectors such as TP1-27, M1-4, and explicit designator lists are accepted.
projectionsDescribe which output actions are generated for a source selection: place a mate part, draw source reference graphics, copy source outline/cutouts, emit PCB labels, and produce related artifacts.

Output

plan writes an MCO JSONC file. run executes that MCO and creates a project with schematic, PCB, copied libraries, placed mating components, net labels, optional source-reference graphics, a user union named DEBUG_PLATE_FEATURES, and requested artifacts such as PCB layer STEP outputs. Generated outputs default under output/debug-plate.

The public example at examples/debug-plate/cricket-node includes a runnable config scaffold, a known-parts manifest seed, expected operations, and input/output folder notes. User-local Altium project files are intentionally not committed.

Tests

Coverage lives in tests/test_debug_plate.py. It verifies template generation, source inspection, selector seeding, known-parts cache metadata, config planning, CLI plan/run behavior, generated MCO shape, labels, part role mappings, source-reference graphics, and STEP artifact operation generation.
