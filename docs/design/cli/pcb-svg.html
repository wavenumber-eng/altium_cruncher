pcb-svg

Renders PCB layer, board, outline, and assembly-view SVG artifacts from Altium PCB documents or projects.

Usage

altium-cruncher pcb-svg board.PcbDoc --export layers --layers top,bottom -o output/pcb-svg
altium-cruncher pcb-svg project.PrjPcb --config pcb-svg.json

Arguments

file accepts .PcbDoc or .PrjPcb. Config and CLI options control selected documents, view sets, export mode, layers, colors, drill rendering, metadata, clipping, scale, assembly projection, and output cleaning.

Assembly Views

assembly-top and assembly-bottom render board outline, side copper, and an optional geometer HLR projection of embedded STEP models. Generated assembly defaults use layer_order: ["copper"]; add silkscreen only when that overlay is intentionally needed.

Board Cutouts

The layer view can emit a synthetic BOARD_CUTOUTS SVG artifact from board-profile cutouts parsed by Altium Monkey. The config fields include_board_cutout_layer, board_cutout_layer_hatch, board_cutout_layer_hash_spacing_mm, board_cutout_layer_hash_angle_deg, board_cutout_layer_hash_line_width_mm, board_cutout_layer_outline_style, board_cutout_layer_outline_dash_mm, board_cutout_layer_outline_width_mm, board_cutout_layer_label, and board_cutout_layer_label_text control whether the red cutout layer is written, whether hashes are drawn through each shape, the hash density, direction, and stroke width, solid versus dashed outlines, outline stroke width, and optional labels. The hash helper fills arbitrary SVG paths, so the same pattern can be reused by future synthetic layers.

Output

The command writes SVG files for configured views plus a sidecar view manifest. Assembly views can produce simple/detail artifacts based on assembly_include_simple and assembly_include_detail.

Tests

L3 verifies Hydroscope PCB layer SVG generation and geometer-backed assembly SVG generation from a copied PcbDoc. Unit tests cover view selection, config behavior, HLR projection caching, and synthetic cutout-layer rendering.
