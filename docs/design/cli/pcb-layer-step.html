pcb-layer-step

Generates a colored STEP model for one selected PCB layer using Altium PCB geometry and geometer planar extrusion.

Usage

altium-cruncher pcb-layer-step board.PcbDoc --layer bottom --copper-color #3D85C6 --outline-color #CCCCCC -o output/step
altium-cruncher pcb-layer-step project.PrjPcb --doc board.PcbDoc --config pcb-layer-step.json

Arguments

file accepts .PcbDoc or .PrjPcb. Options select layer, project context, board document, colors, extrusion thickness, outline behavior, drill-hole handling, polygon inclusion, and fusion. --drill-hole-mode auto is the default. Auto mode uses precise boolean drill cuts for small hole counts and switches to fast drill overlays for dense boards. --drill-hole-mode cut forces exact cutouts, overlay forces visible overlay disks, and none omits drill visualization. --max-boolean-drill-cuts controls the auto threshold.

Output

The command writes a STEP file plus JSON manifest. The manifest records layer, color options, geometry counts, backend, source input, generated filenames, configured drill-hole mode, effective drill-hole mode, boolean drill-cut count, and drill-overlay count. Board cutouts remain boolean cutouts; dense pad/via drill sets may be represented as a thin colored overlay body to keep public CLI runs responsive.

Tests

L3 uses a minimized cricket-node PcbDoc to verify bottom-layer command behavior, color propagation, and dense-board drill overlay request shape. Unit tests cover JSON/JSONC config loading, layer selectors, arc sampling, geometer request shape, primitive preservation, and automatic drill-overlay fallback.
