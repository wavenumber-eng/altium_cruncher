CLI command design

pnp

Exports pick-and-place placement data from Altium projects using the shared BOM/PnP normalization layer for canonical parameter fields, stable sorting, JSON, CSV, and JLC CPL output.

Usage

altium-cruncher pnp project.PrjPcb --format json -o output/pnp
altium-cruncher pnp project.PrjPcb --format csv
altium-cruncher pnp project.PrjPcb --format jlc-cpl

Arguments

file is a .PrjPcb. --format selects csv, json, or jlc-cpl. --units selects mm or mils; jlc-cpl requires mm. --variant, --all-variants, and --exclude-no-bom control variant and component inclusion. Planned options include XLSX, configurable top/bottom grouping, designator prefix ordering, and shared BOM/PnP config policy.

Output

The command writes placement data under the output directory. JSON includes normalized placement records with canonical fields and units. jlc-cpl emits Designator, Layer, Mid X, Mid Y, and Rotation. Planned output should support config-driven naming and shared BOM/PnP filtering rules.

Tests

L3 verifies Hydroscope JSON generation. Unit tests cover placement normalization, top-before-bottom sorting, natural designator sorting, JSON payload shape, and JLC CPL rows. Planned tests should use node_test_array for hierarchy, variant B4, sorting, and JLC CPL behavior.
