CLI command design

bom

Exports bill-of-materials data from Altium schematic or project inputs. The first public version keeps the current CSV, JSON, and XLSX outputs while the richer BOM config and alias system are being designed.

Usage

altium-cruncher bom project.PrjPcb --format json -o output/bom
altium-cruncher bom schematic.SchDoc --format xlsx
altium-cruncher bom project.PrjPcb --variant B4

Arguments

file accepts .SchDoc or .PrjPcb. --format selects csv, json, generic-json, or xlsx. --variant and --all-variants select variant processing. -o selects the output directory.

Output

Current output is written under the selected output directory using the input stem. Planned output includes raw JSON every run, grouped JSON, grouped XLSX, JLC BOM, config-driven field aliases, DNP policy, and optional PCB line-item support.

Tests

Current L3 coverage verifies BOM JSON generation on Hydroscope. Planned BOM parity tests use node_test_array for hierarchy, variants, DNP handling, and field aliasing.
